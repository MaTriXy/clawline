## Web Mapping
Implemented in this pass:

- `/` and `;` open the session popover only when no editable/interactive element owns the key event.
- `Space` and `Enter` focus Prompt Input only when no editable/interactive element owns the key event.
- `Cmd-;` on macOS/iPadOS browsers and `Ctrl-;` elsewhere open the session popover, including while Prompt Input is focused.
- Existing web behavior still owns `Enter` send, `Shift-Enter` newline, and `Escape` composer blur while Prompt Input is focused.

Documented unsupported/conflict cases:

- `Cmd-L`/`Ctrl-L` stays browser-owned for the address bar, so web does not use it for Prompt Input focus.
- `Cmd-,` stays browser-owned where it opens browser settings/preferences.
- `Cmd-=`/`Cmd--`/`Cmd-0` stay browser-owned for zoom.
- `Cmd-J`/`Ctrl-J`, `Cmd-K`/`Ctrl-K`, `Cmd-Shift-J`/`Ctrl-Shift-J`, and `Cmd-Shift-K`/`Ctrl-Shift-K` stay browser/devtools/search/download/text-editing owned, so web does not use them for bubble/chat scrolling.
- `Cmd-.`/`Ctrl-.` is not implemented because web has no exposed cancel-current-prompt UI/control action in Clawline Web yet.
- `Cmd-Shift-H` and `Cmd-Shift-L` are not implemented because they overlap browser/history/home/sidebar/address-bar behavior across Safari/Chromium/Firefox variants.
- Prompt Input `Ctrl-*` editing shortcuts are left to native browser textarea behavior rather than being reimplemented.
