## Product Truth
Web model/status footer parity follows native footer intent from T243/T252 source context:

- Footer/status indicators expose model, thinking/reasoning, and Fast mode through typed session-control actions.
- The controls are not slash-command chat text.
- Model choices come from provider model catalog data when present.
- The footer stays hidden at the normal resting bottom and fades in only when the user scrolls beyond that resting bottom.

Provider seam tickets are not dependencies for this Web ticket unless a live capability mismatch is independently observed.

## Implementation Notes
Web now passes full `/api/session-status` payloads to the selected chat view. `SessionStatusFooter` renders typed controls for:

- `set_model` using `modelCatalog.models` with the current option marked in the picker.
- `set_thinking` / `set_reasoning` based on status display and capabilities.
- `set_fast_mode` / `set_mode` based on status display and capabilities.

Footer reveal is implemented as an extra trailing scroll region: normal scroll-to-bottom stops at the transcript/typing resting bottom, while manual extra scroll reveals the footer by opacity.
