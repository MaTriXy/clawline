## Implementation Notes
Web now passes full `/api/session-status` payloads to the selected chat view. `SessionStatusFooter` renders typed controls for:

- `set_model` using `modelCatalog.models` with the current option marked in the picker.
- `set_thinking` / `set_reasoning` based on status display and capabilities.
- `set_fast_mode` / `set_mode` based on status display and capabilities.

Footer reveal is implemented as an extra trailing scroll region: normal scroll-to-bottom stops at the transcript/typing resting bottom, while manual extra scroll reveals the footer by opacity.
