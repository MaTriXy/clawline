## Product Truth
Web typing indicator visibility mirrors native in-flight behavior:

- `ChatViewModel.swift` owns assistant typing / prompt in-flight state.
- `MessageFlowCollectionView.swift` appends the typing indicator item while assistant generation is in flight.
- `TypingIndicatorCell.swift` defines the rendered typing bubble/dots.

No native/provider ticket is a dependency for this Web ticket.

## Implementation Notes
Web now renders the transcript-bottom typing bubble when either:

- a typed `typing` server payload marks assistant activity active for the session,
- an `event: "activity"` payload reports active assistant work for the session,
- `/api/session-status` reports `run.state` as `running` or `queued`,
- or an assistant message is actively streaming.

Typing state is transient and is scrubbed from persisted chat snapshots.
