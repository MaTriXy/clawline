## Product Truth
Web pairing should mirror native pairing behavior:

- Submit one `pair_request` and keep the pairing WebSocket open while approval is pending.
- Treat `pair_result.reason == "pair_pending"` as transient, not terminal.
- Do not open a new WebSocket and resubmit `pair_request` every 3 seconds during normal pending approval.
- If the pending socket stalls, closes, errors, or times out, require a deliberate retry/resubmission path instead of tight background polling.

## Implementation Notes
Web pairing now keeps the pending socket open after `pair_pending`, updates the UI to awaiting approval, and waits for the terminal `pair_result` on that same socket. The old 3-second pending retry timer was removed.

If the socket closes/errors after a pending result, the UI remains on the awaiting-approval screen with stalled-copy, an error, and a Retry action that performs a deliberate new pairing attempt. Normal pending approval does not show Retry, because the socket is already waiting.
