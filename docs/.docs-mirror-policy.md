# Docs Mirror Policy (intent-based)

Purpose: decide what can be mirrored into contributor-facing `docs/`.

A doc is mirror-eligible only if its frontmatter declares:
- `doc_type: contributor`
- `status: implemented`
- `source_of_truth: code`

Anything else stays out of mirrored contributor docs (proposal/design-note/planned/issue-driven).

## Required frontmatter fields
Every new doc intended for possible mirroring must include:

```yaml
---
doc_type: contributor|design-note|proposal
status: implemented|planned
source_of_truth: code|issue|experiment
---
```

## Rule
Only `contributor + implemented + code` can be mirrored.
