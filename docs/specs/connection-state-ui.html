Connection State UI

Send-button-based connection state indicator. No banners, no input decoration.

States

● Connected

Send button normal, as-is. No changes from current behavior.

● Reconnecting
  • Send button becomes a small yellow pulsing dot.
  • Anchored to the same center point as the normal send button.
  • No icon.
  • Not clickable.
  • User can still type, but cannot send.
● Disconnected
  • Send button turns theme red (design system color TBD — must work in dark + light).
  • Icon changes to a reload symbol.
  • Tapping attempts reconnection → transitions to reconnecting state.
  • User can still type, but cannot send.

Transitions

All state changes use smooth animations. The button morphs between forms — no hard cuts.

  • Connected → Reconnecting: button shrinks and morphs to yellow pulsing dot.
  • Reconnecting → Connected: dot expands back to normal send button.
  • Reconnecting → Disconnected: dot stops pulsing, turns red, reload icon fades in.
  • Disconnected → Reconnecting (on tap): button shrinks to yellow pulsing dot.

What This Is Not

Explicitly excluded:
  • No red banner with dismiss.
  • No color change to the text input border.
  • No error text appearing in the text input field.

Unresponsive State

Not implemented. WebSocket gives clean connected/disconnected/reconnecting signals. "Server is up but slow" is indistinguishable from "big response is coming." False positive cost outweighs value. Three states only.

Design System

A theme red color needs to be defined that works for both dark and light modes. This should go through the design system rather than being hardcoded.

Input Behavior

The text input remains fully functional in all states. Users can type freely. The only thing gated by connection state is the ability to send.
