## Goal

Define the merge doctrine for the next `v2026.4.11` carry-forward under Flynn's frame:

- upstream core is the default truth
- Clawline-owned behavior should live in `extensions/clawline/**` whenever possible
- core carry-forward should be limited to intentionally load-bearing seams
- any file classified as upstream-default must meet a proof obligation:
  - why Clawline still functions on upstream truth
  - what generic/upstream replacement exists, if any
  - what would falsify the claim

This document is the artifact to review, not just a narrative summary.

## Governing Merge Frame

This doctrine is governed by the upstream-merge protocol:

1. Minimize divergence
2. Prefer upstream patterns
3. No invented core hooks
4. Restore rule

Operational meaning in this doctrine:

- any core seam carried forward must be justified as still load-bearing for Clawline function
- if upstream already has a generic seam, adopt that seam and port only the Clawline-required delta
- if the only way to preserve behavior is to keep an obsolete fork structure, stop and escalate rather than silently preserving it
